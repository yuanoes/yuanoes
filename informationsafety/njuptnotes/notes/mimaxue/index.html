# 加密体制的要素

l 明文：发送方将要发送的消息

l 密文：明文被变换成看似无意义的随机消息

l 加密：明文变换成密文的过程称为加密

l 解密：密文恢复出原明文的过程称为解密

l 加密算法：对明文进行加密时所采用的一组规则

l 解密算法：对密文进行解密时所采用的一组规则

l** 加密和解密算法的操作通常都是在一组密钥控制下进行的，分别称为 加密密钥和解密密钥

明文 m

密文 c

加密密钥 k1

加密 Enc (m,k1)

解密密钥 k2

解密 Dec (c,k2)

Dec()

加密算法一般是公开的

密钥管理

对称密码：k1=k2，或密钥间有简单关系，密钥必须保密，用户共享密钥

非对称:k1 已知的情况下，得到 k2 困难的，不同用户享有自己的公私钥对

密码分析：社工手段，穷举密钥空间

安全性：什么样子的密码体制是安全的？

理论安全性：即使攻击者有无限的资源，破译密文也是不可行的。攻击者无论有没有获得密文，对攻击者而言没用。

密钥长度大于等于明文长度：不适用

攻击者在一定时间范围内无法破解密码，密码系统就是有效的

攻击者

  • 唯明文攻击 ：被动攻击的攻击者
  • 已知明文攻击： 获得明文密文
  • 选择性明文攻击 ：获得加密机
  • 选择密文攻击 ：获得解密机

安全目标

  • 有密文不能得到明文，就称密码系统是安全的

  • 不希望能够获得 m 的一些信息

定义：不可区分性

加密系统交互

猜测 cb 是 m0，还是 m1 的密文与加密系统选择一致，攻击者成功。猜对概率为 1/2，攻击者知晓密文概率也不变

# 置换密码

代换密码要先建立一个代换表，密钥即代换表，可以有多个代换表

例如：凯撒密码：a-z 0-25

明文加 3mod26 得到密文非负

最后这里把 3 做为密钥 k 为一个变量进行加密

k 的密钥空间 26，密钥空间过小，暴力破解。

仿射密码

K=K=K=

对任意 m∈M，c∈C，k = (k1，k2)∈K，定义加密变换和相应的解密变换为：
c = ek (m) = k1 m +k2 (mod 26)
m = dk (c) = k1^－1 (c－k2) (mod 26)

当 k1=1 时为一般的凯撒密码

解密则为解同余方程

k1 与 26 互素的个数为 26 的欧拉函数为 12

所以仿射密码密钥的空间量为

12*26=312

密钥空间量也很小

如果把代替表随机匹配

则有 26！种可以有效的避免暴力破解

但是如果密文数够多则可以统计出代替表

该密码体制有一个参数 n。在加解密时，按 n 个字母一组进行变换。
加密变换定义如下：
设密钥 k=(k1,k2,…,kn), 明文 m=(m1,m2,…,mn):
e(k, m)=(c1,c2,…,cn),
其中 ci=(mi + ki)(mod26)，i =1,2,…,n
对密文 c=(c1,c2,…,cn), 解密变换为：
d(k, c)=(m1,m2,…,mn),
其中 mi=(ci －ki)(mod26)，i =1,2,…,n

就是每个移位不一样

# 分组密码

  • 对称密码的加密空间与解密空间相同
  • 维吉尼亚密码不是分组密码
  • ** 分组密码的思想：** 将明文消息编码表示后的数字序列划分成长为 n 的组，各组分别在密钥 k 控制下变换成等长的输出数字序列
  • 通常由一个简单的函数 F 迭代若干次而形成，而 F 都是基于代换 (S 盒)— 置换 (P 盒) 的网络，S 盒的设计是分组密码的核心，s 实现混肴的作用，f 实现扩散的作用

f 函数非线性，不一定可逆
