# ctf 一道无字母数字解题思路：

# 0x00

打开题目如图：

# 0x01

首先观察到有 eval 函数可以把字符串作为 PHP 代码执行，所以只要绕过过滤利用这个函数就能 websell。

然后可以看到的过滤有字符长度和字符限制

preg_match 函数执行的是对变量 a 的正则表达式匹配，可以看到没有被过滤的字符有，a？/><` ' +，所以开始的思路是利用无字母数字 webshell (结果根本绕不过)

# 0x02

百度搜索到了离别歌的一篇无字母数字 webshell 文章，可以发现主要有俩种方式来绕过

php 里有 'a'++ => 'b'，'b'++ => 'c'... 所以，只要利用 a，通过自增操作即可获得 a-z 中所有字符。

但是 有长度限制，考虑用数组来绕过 strlen 函数，但定义变量的_也被过滤了，遂放弃。

# 0x03

正解方法是利用通配符来绕过，用？来匹配任意字符，也就是说？？？/??? =bin/cat

这里补充一下通配符的知识点:

在计算机（软件）技术中，通配符可用于代替单个或多个字符。[1] 通常地，星号 “*” 匹配 0 个或以上的字符，问号 “?” 匹配 1 个字符。

# 文件和目录 [编辑]

在 CP/M、DOS、Microsoft Windows 和类 Unix 操作系统中描述文件名 (或路径) 时，星号（ * ）匹配零个或多个任意字符、问号（ ? ）匹配零个或不多于问号个数的任意字符。如： 123??? 将匹配 1231 或 12313 ，但不会匹配 123991991 。

在 Unix shell 和 Windows PowerShell 中，方括号（ [ 和 ] ）括起来的字符范围匹配在此范围内的任意字符，例如： [A-Za-z] 匹配任意的大小写字母。Unix shells 可以在用括号括起来的指定字符前加一个 "!" 来表示否定。匹配到多个文件或路径的通配符模式称为 glob 扩展。

这里还有一个知识点就是 php 短标签的问题，一般情况下短标签是默认开启的

如果设置为 Off，则不能正常解析类似于这样形式的 php 文件：
<?
phpinfo()
?>

而只能解析
<?php
phpinfo()
?>
这样形式的 php 文件
所以要想 php 支持短标签，需要我们把 short_open_tag

设置为 On.

所以构造 payload 为 'a=?><?= /???/?a? /??a? '?>// 反引号是用来执行 bin/cat flag 命令，正好绕过 echo 函数，再用两个引号闭合即可
